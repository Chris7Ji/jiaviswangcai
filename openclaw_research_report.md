## 一、OpenClaw的起源

### 1.1 创始人/团队背景

**核心创始人**: Peter Steinberger
- PSPDFKit创始人（知名的PDF处理SDK公司）
- 资深iOS/macOS开发者
- 在开发者社区有较高知名度

### 1.2 项目演进时间线

| 时间 | 事件 | 说明 |
|------|------|------|
| 2025年11月 | 项目首次发布 | 最初名为 **Clawdbot** |
| 2025年底 | 第一次更名 | 更名为 **Moltbot** |
| 2026年初 | 正式定名 | 更名为 **OpenClaw** |
| 2026年1月 | 爆发式增长 | GitHub Star数快速攀升 |
| 2026年1月30日 | 安全修复 | 发布v2026.1.29修复CVE-2026-25253漏洞 |
| 2026年2月 | 社区生态建立 | ClawHub技能市场活跃 |
| 2026年3月 | 国内爆发 | 腾讯云、阿里云推出预装镜像 |

### 1.3 最初的产品形态

OpenClaw最初定位为**个人AI助手运行时框架**，核心设计理念：
- 在用户自己的设备上运行（本地优先）
- 连接多种消息平台（WhatsApp、Telegram等）
- 提供7×24小时AI助手体验
- 支持多Agent协作和技能扩展

### 1.4 技术架构演进

**早期架构**:
- 单一Gateway控制平面
- 基于WebSocket的通信
- 支持有限的消息渠道

**当前架构**:
```
消息渠道层 → Gateway控制平面 → Agent层 → Skills层
                ↓
            Nodes层（iOS/Android设备端点）
```

**核心组件**:
- Gateway: 单一控制平面，localhost:18789
- Agent: LLM决策引擎（支持Claude/GPT/Ollama等）
- Skills: JS/TS可扩展工具包
- Channels: 统一消息接口
- Nodes: 设备传感器/端点

## 二、OpenClaw为什么这么火

### 2.1 核心创新点（vs 传统AI助手）

| 维度 | 传统AI助手 | OpenClaw |
|------|-----------|----------|
| 部署方式 | 云端SaaS | 本地自托管 |
| 数据隐私 | 数据上传云端 | 数据本地保留 |
| 执行能力 | 被动问答 | 主动执行任务 |
| 平台接入 | 单一应用 | 20+消息平台 |
| 扩展性 | 固定功能 | 技能插件生态 |
| 定制化 | 有限 | 完全可编程 |

**三大核心能力**:
1. **Agent-Browser**: 智能网页操作，可自主完成点击、滚动、数据提取
2. **Daily-Report**: 自动化日报生成，整合Git提交、协作工具动态
3. **零配置启动**: Docker镜像预置全套工具链

### 2.2 解决的痛点问题

**用户痛点**:
- 现有AI工具只能"你问我答"，无法主动执行任务
- 数据隐私担忧（云端AI需要上传个人数据）
- 多平台割裂（需要在不同应用间切换）
- 开发者想要可编程、可定制的AI助手

**OpenClaw的解决方案**:
- ✅ 主动执行：定时任务、Cron作业、Webhook触发
- ✅ 数据本地：Gateway运行在本地设备
- ✅ 统一入口：WhatsApp/Telegram/飞书等都可接入
- ✅ 开源可扩展：MIT协议，支持自定义Skills

## 四、技术亮点

### 4.1 Skills生态系统

**架构设计**:
- 基于SKILL.md的声明式配置
- 支持JS/TS编写自定义技能
- 三层技能优先级：Workspace > Local > Bundled

**技能分类**（精选5490+技能）:
| 类别 | 数量 | 示例 |
|------|------|------|
| Coding Agents & IDEs | 1222 | 代码辅助、IDE集成 |
| Web & Frontend Dev | 938 | 前端开发工具 |
| DevOps & Cloud | 409 | 云服务操作 |
| Search & Research | 350 | 搜索、研究 |
| Browser & Automation | 335 | 浏览器自动化 |
| Git & GitHub | 170 | Git操作 |
| Image & Video | 169 | 媒体处理 |

**ClawHub技能市场**:
- 官方注册表：clawhub.ai
- 支持版本管理、标签、搜索
- 集成VirusTotal安全扫描

### 4.2 Agent协作机制

**多Agent路由**:
```
Inbound Message → Gateway → 路由规则 → 对应Agent
```

**协作模式**:
- 主Agent（main）处理主要对话
- 子Agent处理特定任务
- Agent间可以spawn子任务
- 支持工作空间隔离

**会话管理**:
- 独立会话上下文
- 会话重置策略（节省Token）
- 跨会话记忆持久化

### 4.3 记忆系统创新

**三层记忆架构**:
1. **会话记忆**: 当前对话上下文
2. **工作区记忆**: 项目相关的持久化记忆
3. **系统记忆**: 跨项目的全局记忆（SOUL.md）

**SOUL.md机制**:
- 类似AI的"灵魂"配置文件
- 定义AI的人格、偏好、工作方式
- 支持onlycrabs.ai分享和复用

**记忆持久化**:
- 本地文件存储
- 支持多种记忆后端插件
- 可配置的隐私级别
