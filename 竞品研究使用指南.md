## 💡 使用方法

### 1. 快速市场扫描 (Quick Scan)
**时间**: 15-30分钟  
**适用**: 初步探索、验证想法

**触发方式**:
```
研究[市场/细分领域]的竞品
```

**示例**:
- "研究AI Agent市场的竞品"
- "看看有哪些竞品在做AI音乐生成"
- "快速扫描企业级搜索工具的竞品"

**输出**:
- 5-10个主要玩家列表
- 一句话描述每个竞品
- 市场主导者观察

### 2. 标准竞品分析 (Standard)
**时间**: 1-2小时  
**适用**: 商业规划、市场进入决策

**触发方式**:
```
分析[细分领域]的竞品格局
```

**示例**:
- "分析AI写作工具的竞品格局"
- "研究企业级聊天机器人的市场竞争"
- "分析低代码平台的竞品策略"

**输出**:
- 完整市场格局
- 竞品对比矩阵
- 3-5个识别出的市场机会
- 推荐的定位角度

### 3. 深度竞品审计 (Deep Dive)
**时间**: 半天+  
**适用**: 严肃竞争、详细战略规划

**触发方式**:
```
深度分析[公司名]的竞品策略
```

**示例**:
- "深度分析Perplexity的产品策略"
- "全面审计Notion的竞品定位"
- "详细研究Figma的设计工具策略"

**输出**:
- 单个竞品深度分析
- 详细SWOT分析
- 战略行动手册

### 4. 功能对比分析
**触发方式**:
```
对比[竞品A]、[竞品B]、[竞品C]的[某方面]
```

**示例**:
- "对比OpenAI、Anthropic、Google的AI产品功能"
- "比较Midjourney、DALL-E、Stable Diffusion的图像生成能力"
- "对比Suno、Udio、AIVA的音乐生成功能"

### 5. SEO/GEO分析
**触发方式**:
```
分析[竞品]的SEO策略
```

**示例**:
- "分析竞品的SEO策略和关键词排名"
- "看看竞争对手如何在AI搜索中被引用"
- "分析HubSpot的营销关键词策略"

## 📊 分析框架

### 竞品档案结构
每个竞品分析包含：

**BASICS (基础信息)**
- 一句话描述做什么
- 目标客户
- 定价模式和范围
- 成立时间、融资、规模指标

**PRODUCT (产品)**
- 核心功能
- 关键差异化点
- 弱点/缺失
- 最近变化

**POSITIONING (定位)**
- 如何描述自己
- 与谁对比
- 信息传递的语气和风格

**TRACTION SIGNALS (增长信号)**
- G2/Capterra评分和评价
- 强调的社会证明
- 客户logo/推荐
- 增长指标

### 竞争矩阵
对比维度包括：
- 价格 ($/$$/$$$)
- 核心功能 ✅/❌
- 目标用户
- 差异化优势
- 技术架构
- 客户评价

### 差距分析
识别：
- 客户抱怨但无人解决的问题
- 被忽视的用户群体
- 定价过高的市场空白
- 缺失的功能

## 🎯 研究流程

### 第一次会话：市场格局
```
"我想研究[细分领域]的竞品"
→ 快速扫描市场
→ 识别5-10个关键玩家
→ 创建市场概况
→ 询问：想深度分析哪个竞品？
```

### 后续会话：深度分析
```
"分析[公司X]"
→ 加载市场概况获取背景
→ 完整竞品分析
→ 保存到 niches/{niche}/{company}.md
→ 更新市场概况
```

### 回访
```
"了解[细分领域/公司]的情况"
→ 加载现有研究
→ 标记可能过时的信息
→ 提供刷新特定部分
```

## ⚠️ 注意事项

### 数据准确性
- **标注来源**: 每个发现都标注来源
- **标注日期**: 价格、功能会变化
- **置信度**: High(直接来源)/Medium(推断)/Low(猜测)

### 隐私和伦理
- ✅ 只使用公开信息
- ✅ 不创建虚假账户
- ✅ 不违反服务条款
- ❌ 不访问私有系统
- ❌ 不窃取商业机密

### 常见陷阱
- **无明确范围** → 总是先明确研究目的
- **过度关注功能** → 商业模式和定位往往更重要
- **过时定价** → 直接检查定价页面
- **忽视替代品** → 直接竞品不是唯一威胁
- **分析瘫痪** → 设定时间限制，完成比完美重要
- **无建议** → 竞品列表不是策略，要有行动建议
