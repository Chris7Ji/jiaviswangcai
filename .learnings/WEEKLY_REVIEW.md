# 每周回顾检查清单
# 执行频率: 每周日
# 执行者: AI助理旺财
# 指示来源: 老板（季）

## 📋 每周回顾任务

### 1. 检查LEARNINGS.md
- [ ] 统计本周新增学习记录数量
- [ ] 识别重复模式（出现3次以上的类似问题）
- [ ] 提取核心学习点（可提升到SOUL.md/AGENTS.md的）
- [ ] 标记已验证的学习（实际应用后确认有效）

### 2. 检查ERRORS.md
- [ ] 统计本周新增错误记录
- [ ] 分析错误类型分布（工具失败/理解错误/执行错误）
- [ ] 检查是否有重复错误（需要重点解决的）
- [ ] 确认已修复的错误状态

### 3. 检查FEATURE_REQUESTS.md
- [ ] 查看新的功能请求
- [ ] 评估可行性和优先级
- [ ] 标记已实现的功能

### 4. 模式提取与提升
对于以下情况，提升到核心记忆文件：

**提升到SOUL.md的情况**:
- 行为模式改变（如"老板偏好简洁回复"）
- 核心原则确认（如"我是老板的AI助理"）
- 边界定义（如"不执行删除操作"）

**提升到AGENTS.md的情况**:
- 任务派发规则优化
- Agent协作流程改进
- 新Agent配置需求

**提升到TOOLS.md的情况**:
- 工具使用技巧
- 环境配置变更
- 新工具集成方法

### 5. 执行归档
如果记录数量超过10条:
- [ ] 执行归档脚本: `bash ~/.openclaw/workspace/.learnings/archive/auto_archive.sh`
- [ ] 验证归档文件已创建
- [ ] 确认活跃记录已精简

### 6. 向老板汇报
每周回顾完成后，向老板汇报：
- 本周学习记录数量
- 发现的重要模式
- 已提升的核心记忆
- 需要关注的问题
