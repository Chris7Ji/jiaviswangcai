# 创建时间: 2026-03-10
# 指示来源: 老板（季）

## 🧠 记忆系统架构

### 三层记忆体系
1. **会话级记忆** - 当前对话上下文（临时）
2. **项目级记忆** - 持久化存储（MEMORY.md, .learnings/）
3. **技能级记忆** - 技能配置（SKILL.md, AGENTS.md, SOUL.md）

### 核心文件位置
```
~/.openclaw/workspace/
├── MEMORY.md                 # 核心长期记忆
├── AGENTS.md                 # Agent团队配置
├── SOUL.md                   # 身份、性格、边界
├── USER.md                   # 老板信息
├── TOOLS.md                  # 工具配置
├── HEARTBEAT.md              # 定期检查清单
└── .learnings/               # 学习记录目录
    ├── LEARNINGS.md          # 学习记录
    ├── ERRORS.md             # 错误记录
    ├── FEATURE_REQUESTS.md   # 功能请求
    ├── WEEKLY_REVIEW.md      # 每周回顾检查清单
    ├── MONTHLY_ARCHIVE.md    # 每月归档检查清单
    ├── QUARTERLY_REVIEW.md   # 每季审查检查清单
    └── archive/              # 归档目录
        ├── auto_archive.sh   # 自动归档脚本
        ├── weekly/           # 每周归档
        ├── monthly/          # 每月归档
        └── quarterly/        # 每季归档
```

## 📝 记录规范

### 何时记录到LEARNINGS.md
- ✅ 用户明确说"记住..."
- ✅ 用户纠正我的错误
- ✅ 发现更好的工作方法
- ✅ 成功解决复杂问题

### 何时记录到ERRORS.md
- ✅ 命令/操作失败
- ✅ 外部API/工具失败
- ✅ 我的知识过时/错误

### 何时记录到FEATURE_REQUESTS.md
- ✅ 用户需要新功能
- ✅ 发现缺失的能力

### 记录格式
```markdown
## YYYY-MM-DD HH:MM:SS
**ID**: learn-XXX / error-XXX / feature-XXX
**优先级**: CRITICAL / HIGH / MEDIUM / LOW
**类别**: [correction|knowledge_gap|best_practice|...]
**摘要**: [一句话总结]
**场景**: [详细描述]
**核心要点**: [关键点]
**应用**: [如何应用]
**状态**: ✅ 已验证 / ⏳ 待验证
```

## 🔄 维护流程

### 每周日（每周回顾）
1. 检查LEARNINGS.md新增记录
2. 识别重复模式（出现3次以上）
3. 提取核心学习点
4. 如记录>10条，执行归档
5. 向老板汇报

**执行命令**:
```bash
bash ~/.openclaw/workspace/.learnings/archive/auto_archive.sh
```

### 每月末（每月归档）
1. 将当月记录归档到archive/monthly/
2. 生成月度摘要
3. 清理活跃记录（保留≤10条）
4. 提取月度核心模式

### 每季度末（每季审查）
1. 全面审查MEMORY.md（删除过时信息，确保≤100行）
2. 全面审查AGENTS.md（更新Agent配置）
3. 全面审查SOUL.md（确认核心原则）
4. 全面审查TOOLS.md（更新工具配置）
5. 季度总结归档

## 🚨 异常情况处理

### 记忆检索超时
- **现象**: memory_search 超过300秒
- **处理**: 直接读取文件，跳过语义搜索
- **记录**: 记录到ERRORS.md

### 记忆文件损坏
- **现象**: 文件无法读取或格式错误
- **处理**: 从备份恢复或重建
- **预防**: 定期备份到archive/

### 记忆容量超限
- **现象**: MEMORY.md > 100行
- **处理**: 归档旧内容，提取核心模式
- **预防**: 每周检查，及时归档

## 💡 最佳实践

1. **及时记录**: 重要信息立即记录，不要拖延
2. **定期回顾**: 每周回顾学习记录，提取模式
3. **主动归档**: 不要等满了再归档，保持活跃记录精简
4. **验证应用**: 记录后要实际应用，验证有效性
5. **向老板汇报**: 每周回顾后向老板汇报重要发现
