### 3. 命令执行安全 🟡🟡 (高风险)

**当前配置:**
- alsoAllow条目数: **105条**
- strictInlineEval: null (未设置)

**允许执行的命令分类:**

| 类别 | 条目数 | 风险等级 |
|------|--------|----------|
| 路径限定脚本 | ~15 | 低 |
| 系统命令(node/git/curl等) | ~90 | **高** |

**高风险命令:**

```bash
# 网络相关 - 可发起任意HTTP请求
curl -s      # 无限制网络访问
curl -X      # 任意HTTP方法
curl -H      # 任意HTTP头
curl -d      # 发送任意数据
curl -o      # 下载任意文件

# 系统命令 - 权限过大
node         # 任意Node.js代码执行
python3      # 任意Python代码执行
bash         # 任意bash脚本
sh           # 任意shell脚本

# 通用命令
kill         # 终止任意进程
killall      # 批量终止进程
```

**业界最佳实践:**
| 建议 | 当前状态 |
|------|----------|
| 仅允许特定脚本路径 | ❌ 允许裸命令 |
| 禁止裸curl/wget | ❌ 允许 |
| 禁止裸node/python3 | ❌ 允许 |
| strictInlineEval=true | ⚠️ 未设置 |
| 命令超时限制 | ❌ 无限制 |

**建议:**
1. 设置 `strictInlineEval: true`
2. 移除所有裸命令条目(node, python3, bash, sh, curl等)
3. 仅保留必要的脚本路径
4. 考虑使用路径前缀匹配而非完整路径
