# 成长日记生成规范 (DIARY_INSTRUCTIONS.md)

> 本文件指导每日 21:00 成长日记的生成规范
> 最后更新：2026-04-17（重大修订：强调诚实原则）

---

## ⚠️ 核心原则（最高优先级）

### 第一原则：绝对不虚构内容

**绝对禁止生成任何不存在的内容。** 这是工作的第一原则，无论任何情况都不能违反。

- 不知道今天做了什么 → 诚实写"今天没有找到具体的活动记录"
- 不确定某个数字 → 不要编造，使用"约"或"待确认"
- 遇到信息缺失 → 如实标注"内容待验证"

### 第二原则：优先记录实际活动

日记应该记录**我（AI Agent 总指挥旺财Jarvis）今天实际做了什么事情**，而不是编造技术工作汇报。

---

## 日记内容要求

### 应该记录的内容

- 定时任务（cron）的执行状态（成功/失败/超时）
- 系统发生的事件（崩溃、重启、bug修复）
- 网站或配置的重大变更
- 收到的指令和完成的工作
- 发现的系统问题及解决方案
- 定时邮件/通知的发送状态

### 禁止的内容

- 虚构的技术工作（如"昇腾模型迁移基准测试"、"RDMA通信优化"等）
- 不存在的命令、脚本、配置参数
- 编造的性能指标（"性能提升15%"、"延迟降低8ms"等）
- 不属于自己的工作（如昇腾工程师的技术工作）

## 日记结构模板

```html
<h2>今日事件概述</h2>
<p>【一句话总结今日主要事件】</p>

<h2>一、系统/任务状态</h2>
<p>【记录今日系统的主要事件，如cron任务执行状态、重启事件等】</p>
<ul>
<li>✅ 【任务名称】：成功执行</li>
<li>❌ 【任务名称】：失败/超时（原因）</li>
<li>⚠️ 【任务名称】：部分成功（具体情况）</li>
</ul>

<h2>二、完成的实际工作</h2>
<p>【记录今天实际完成的工作，如修复了什么、管理了什么配置等】</p>

<h2>三、遇到的问题</h2>
<p>【记录今天遇到的实际问题】</p>

<h2>四、后续计划</h2>
<p>【如果有明确的待办事项】</p>
```

---

## 内容诚实性检查

在生成日记前，请自问：

1. **这件事我真的知道吗？** → 如果不确定，不要写
2. **这是我实际做的吗？** → 不要写别人做的事情
3. **这个数字准确吗？** → 如果不准确，不要写或标注"约/待确认"
4. **这个描述符合事实吗？** → 如果不符合，修正或删除

## 示例

### 好的例子（诚实）：

```html
<h2>今日事件概述</h2>
<p>今日主要处理了系统重启和成长日记数据恢复工作。</p>

<h2>一、diary.js恢复</h2>
<p>从git历史恢复了17条丢失的日记（3/18-4/3）。</p>
```
